## Największe luki vs architektura docelowa (podsumowanie agenta)

1. **No public storefront**: the spec's landing (hero, Monitor demo, plan comparison, FAQ, footer — §6) doesn't exist; production root IS the flow, and an unrouted light `LandingPage.tsx` sits dead in the repo.
2. **Home is temperature-first, spec is machine-first**: the six-mode picker forces Home/demo users to choose −11/−12/−13/Ninja modes; §4.1/§8's one-time machine onboarding, machine catalogue (§9), and "flavor → amount → recipe" repeat flow are entirely missing.
3. **Monitor as product signature doesn't exist**: no Monitor icon, no drawer/bottom-sheet, no 1–10 headline, no 4-trait Home view (§7.1, §13); today it's an inline tuning section whose primary action is permanently blocked on production.
4. **Dark vs light — now decided**: current customer shell, classic shell and Studio are all dark; per the owner decision light-first is binding, so this is a confirmed P0 gap, with the customer shell cheap to revert (delete the `customerDarkVars` remap/DarkShell) and the classic/Studio `bg-shell text-ivory` surfaces the expensive half.
5. **Score model wrong on both surfaces**: Studio shows `/100`, customer result shows no score at all — §15's 1–10 "Dopasowanie receptury" verdict layer is unbuilt.
6. **Temperature discipline broken exactly where it matters**: Studio offers −14/−18 as serving and omits −13, while `storage-minus-18` lives inside the serving-profile enum — §11's serving/storage split needs a data-level fix, not copy patches.
7. **Production is a shop-window with locked doors**: no sign-in, no saving, no catalogue, no checkout — every flow ends in an "unavailable in this build" style dead end, often in internal language.
8. **Pro Studio is one wall, not modules**: ~17 equal panels + dash-placeholder previews vs §14's grouped, collapsible, pinnable Monitor Pro; §17 gram-locks and §18 conflict explanations have no UI at all.
9. **Copy/system hygiene**: EN/PL patchwork, internal codes (IF9/IF10, fixture names), tiny 10px uppercase labels and sub-AA muted text on dark — all §21 violations that the light re-skin should fix in one pass.
10. **What's already right**: honest no-fabrication guarantees, 44–56px touch targets, no horizontal scroll at 390px, desaturated status tokens (no red-green-red), reduced-motion handling and a solid focus-ring system in the customer shell — the §21 foundations exist and are worth carrying into the light redesign.
